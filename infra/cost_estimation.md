# Meeting Minutes — AWS コスト見積もり

リージョン: `ap-northeast-1`（東京） | 料金モデル: オンデマンド | 通貨: USD

すべての料金は 2026年4月時点の `ap-northeast-1` における [AWS Pricing API](https://docs.aws.amazon.com/awsaccountbilling/latest/aboutv2/price-changes.html) から取得しています。Bedrock の料金は [Amazon Bedrock 料金ページ](https://aws.amazon.com/bedrock/pricing/) を参照しています。

## 使用量の前提条件

以下の見積もりは**小規模チーム MVP シナリオ**に基づいています：

| パラメータ | 値 |
|-----------|-----|

| 平均レポートサイズ | 3 KB JSON |
| 月間フロントエンドページビュー | 2,000 |
| フロントエンドアセットサイズ | 合計約5 MB |

## サービス別コスト内訳

### 1. Amazon Transcribe Streaming

Transcribe Streaming は処理された音声の秒数で課金されます。

| 指標 | 値 |
|------|-----|
| 単価 | $0.0004/秒（最初の250K分まで） |
| 会議あたりの音声 | 1,800秒 |
| 月間会議数 | 100 |
| 合計秒数 | 180,000 |
| **月額コスト** | 180,000 × $0.0004 = **$72.00** |

> Transcribe Streaming が最大のコスト要因です。使用量が少ない場合（例：月20会議）、約$14.40に下がります。

### 2. Amazon Bedrock（Claude 3 Haiku）

Bedrock はオンデマンド推論の入力/出力トークンごとに課金されます。

| 指標 | 値 |
|------|-----|
| 入力トークン単価 | $0.00025 / 1Kトークン |
| 出力トークン単価 | $0.00125 / 1Kトークン |
| 会議あたりの入力トークン | 約5,000（トランスクリプト）+ 約1,000（プロンプト）= 6,000 |
| 会議あたりの出力トークン | 約2,000（レポート JSON） |
| 月間会議数 | 100 |
| 合計入力トークン | 600,000 |
| 合計出力トークン | 200,000 |
| 入力コスト | 600 × $0.00025 = $0.15 |
| 出力コスト | 200 × $0.00125 = $0.25 |
| **月額コスト** | **$0.40** |

> Claude 3 Haiku を使用した Bedrock は、構造化抽出タスクに非常にコスト効率が良いです。

### 3. AWS Lambda

Lambda はリクエストごとと GB 秒あたりのコンピューティングで課金されます。

| 指標 | 値 |
|------|-----|
| リクエスト単価 | $0.20 / 100万リクエスト |
| コンピューティング単価 | $0.0000166667 / GB秒 |
| 会議あたりのリクエスト | 約15（WebSocket メッセージ + ワークフローステップ + API コール） |
| 月間合計リクエスト | 100 × 15 = 1,500 |
| リクエストあたりの平均時間 | 2秒 |
| 平均メモリ | 0.5 GB（全 Lambda の加重平均） |
| 合計 GB 秒 | 1,500 × 2 × 0.5 = 1,500 |
| リクエストコスト | 1,500 × $0.0000002 = $0.0003 |
| コンピューティングコスト | 1,500 × $0.0000166667 = $0.025 |
| **月額コスト** | **$0.03** |

> Lambda の無料枠には月100万リクエストと400,000 GB秒が含まれます。この規模では Lambda は実質無料です。

## コスト最適化の推奨事項

1. **Transcribe がコストの96%** — リアルタイムが不要な場合はバッチ文字起こし（ストリーミングの $0.024/秒 に対して $0.024/分）を検討するか、キャプチャする会議時間を短縮してください。

2. **無料枠のカバー範囲** — Lambda、Step Functions、Cognito、DynamoDB はこの規模では実質無料です。S3 も無視できるレベルです。

3. **CloudFront** — データ転送の月1 TB 無料枠が小規模チームのほとんどの使用をカバーします。それを超えた場合のみコストが発生します。

4. **Bedrock モデルの選択** — Claude 3 Haiku は最もコスト効率の良い選択肢です。より大きなモデル（Sonnet/Opus）に切り替えると、コストが10〜50倍に増加します。

5. **リザーブドキャパシティ** — サーバーレスサービスには適用されません。すべてのサービスが従量課金です。

6. **ライフサイクルポリシー** — 90日後の S3 Glacier 移行により、古いトランスクリプトとレポートの長期ストレージコストが削減されます。

## アイドル時のコスト（使用していない時）

アプリがデプロイされているが積極的に使用されていない場合、継続的なコストは以下のみです：

| リソース | アイドル時コスト |
|---------|----------------|
| S3 ストレージ | 保存 GB あたり月約$0.01 |
| CloudWatch ログストレージ | 保存 GB あたり月約$0.03 |
| CloudFront ディストリビューション | $0.00（アイドル時は課金なし） |
| その他すべてのサービス | $0.00（従量課金のみ） |

サーバーレスアーキテクチャにより、アプリがアイドル状態の時はほぼコストがかかりません。

## 免責事項

これらの見積もりは、上記の使用量前提条件と 2026年4月時点の `ap-northeast-1` における公開 AWS 料金に基づいています。実際のコストは、実際の使用パターン、データ転送量、AWS 料金の変更によって異なる場合があります。具体的なワークロードに基づくより正確な見積もりには、[AWS 料金計算ツール](https://calculator.aws/) をご利用ください。
